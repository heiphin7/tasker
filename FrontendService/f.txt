Все реализовано через kafka, и есть 3 основных сервиса:
Kafka поднята на Docker

FrontendService:
    Именно данный сервис отвечает за обработку получения всех страниц и также он делает запросы на другие сервисы
    для получения данных, которые отображаются на странице. Также данный сервис отвечает за Авторизацию и Регистрацию,
    так как выделять отдельный сервис под эти 2 задачи будет плохо, так как чтобы получить 1 страницу, нужно будет
    ждать пока 2-4 запроса обработатся, что очень долго
TaskService:
UserService:

Регистрация/Авторизация (FrontendService)

- Регистрация (С проверкой уникальности Username & Email)
- Вход в акканут (База)
- Авторизация и Аутентификация с Jwt
- Logout

Личные задачи (TaskService)

- CRUD:
    Создание,
    Редактирование,
    Удаление
    Получение задач

Команды (UserService):

- Создание команды с указанием пользователей (Реализована проверка, что все указанные пользователи существуют и обрабатывается случай когда нет)
- Team Details Страница
- Админство (Тут реализовано огромное количество проверок, то есть только у админа есть кнопки для управления командой
            Например кнопка создать задау, добавить участника, удалить участник видна и доступна только АДМИНУ!)
- Передача прав админа
- Функция покинуть команду (Если это учатсник просто, то он смело покинент, а если это админ то он должен оставить "наследника")
- Добавление / Удаление участника (С проверкой прав)
- Создание Командных задач с выбором Исполнителя
- Изменения Командных задач с проверкой прав доступа
- Удалеение Командной задачи (Кнопка доступно только и только администратору)
- Удаление пользователей с проверкой прав доступа


- ФУНКЦИЯ ПОДЗАДАЧ:
- У каждой командной задачи есть модальное окошко и там можно создавать подзадачи к задаче
- Создание подзадач доступно только администратору
- Удаление и изменение подзадач с проверкой доступа и прав
